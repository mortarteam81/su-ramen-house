# 🍜 라면가게 (su-ramen-house)

슈의 라면가게에서 영감을 받은 **라면 조리 웹 게임**입니다.  
손님의 주문을 확인하고, 냄비를 선택한 뒤, 레시피 순서대로 재료를 넣어 라면을 완성해 서빙하세요!

---

## 🎮 게임 특징

- **주문/서빙 기반 타이밍 게임**
- **레시피 조합 시스템**
  - 재료를 넣는 도중 가능한 레시피 후보를 힌트로 보여주고, 특정 레시피로 “조리 확정” 가능
- **콤보 보너스**
  - 연속 서빙 성공 시 콤보가 쌓이고, 일정 콤보마다 추가 보너스를 획득
- **메뉴 해금 시스템**
  - 게임 중 번 돈(세션 수익) 기준으로 메뉴가 자동 해금되거나,
  - “📖 메뉴 도감(상점)”에서 소지금으로 해금 가능
- **로컬 저장(LocalStorage)**
  - 소지금/최고기록/해금 메뉴 등이 브라우저에 저장됨

---

## 🕹️ 플레이 방법 (요약)

1. 메인 메뉴에서 **게임 시작**
2. 손님이 오면 **주문 확인**
3. **빈 냄비 클릭** → 냄비 선택
4. 재료 선반에서 **재료를 레시피 순서대로 투입**
5. 조리 완료 후 **서빙하기**로 주문한 손님에게 전달
6. 손님의 인내심이 다하기 전에 서빙하세요!  
   실패가 누적되면 게임 오버입니다.

---

## 🍜 레시피 / 재료

프로젝트 내 레시피/재료 정의는 `js/config.js`에 있습니다.

### 재료(예시)
- 💧 물, 🍜 면, 🧂 스프, 🥚 계란, 🥬 김치, 🦐 해물, 🧀 치즈, 🍡 떡, 🌶️ 고추

### 메뉴(예시)
- 🍜 기본 라면: `water → noodle → soup`
- 🥚 계란 라면: `water → noodle → soup → egg`
- 🥬 김치 라면: `water → noodle → soup → kimchi`
- 🌶️ 매운 라면, 🍡 떡라면, 🧀 치즈 라면, 🦐 해물 라면, ⭐ 스페셜 라면 등

## 🚀 실행 방법

이 프로젝트는 별도 빌드 없이 **정적 파일**로 실행됩니다.

### 1) 로컬에서 실행
1. 이 레포를 클론/다운로드
2. 브라우저에서 `index.html`을 열기

> 참고: 일부 브라우저/환경에서는 ES Module(`type="module"`) 로딩 때문에
> 파일 직접 열기(`file://`)가 제한될 수 있습니다. 이 경우 간단한 로컬 서버로 실행하세요.

### 2) 간단 로컬 서버(권장)
아래 중 하나로 실행 후 브라우저에서 접속:

- Python
  - `python -m http.server 8000`
- Node (npx)
  - `npx serve .`

## 🧱 프로젝트 구조

- `index.html` : 화면(메뉴/게임/상점/게임오버) 레이아웃
- `css/`
  - `style.css` : 공통 스타일
  - `game.css` : 게임 화면 UI 스타일
  - `animations.css` : 애니메이션 효과
- `js/`
  - `main.js` : 엔트리/이벤트 바인딩/화면 전환
  - `game.js` : 게임 상태/루프/서빙/일시정지/저장 처리
  - `config.js` : 재료, 레시피, 고객 타입, 밸런스 상수
  - `cooking.js` : 냄비/조리 로직
  - `customer.js` : 고객 스폰/인내심/주문 관리
  - `ui.js` : DOM 업데이트, 토스트/이펙트, HUD 갱신
  - `menu.js` : 메뉴 해금 관리
  - `storage.js` : LocalStorage 저장/불러오기

## 💾 저장 데이터

브라우저 LocalStorage에 저장됩니다(키: `ramen_shop_save`).  
저장되는 값 예시:
- 소지금
- 최고 기록(세션 수익 기준)
- 최고 콤보
- 해금된 메뉴 목록
- 누적 서빙 수 등
